# NewLife.XCode 架构设计

> 版本：v12.0 | 日期：2026-07-15

本文档描述 NewLife.XCode 的整体架构分层、核心组件及其协作关系。功能清单见[功能模块清单](功能模块清单.md)，需求定义见[需求文档](需求文档.md)。

---

## 1. 总体架构

XCode 采用**模型驱动 + 运行时元数据 + 多级缓存 + 可选分片**的分层架构，从上到下分为四层：

```
┌─────────────────────────────────────────────────┐
│                  应用层 (Application)              │
│  Entity<T>.Find/FindAll/Insert/Update/Delete     │
│  IQueryable<T> LINQ / DAL.Select<T>()             │
│  DbService HTTP 客户端                            │
├─────────────────────────────────────────────────┤
│              查询 DSL 层 (Query DSL)               │
│  WhereExpression / IQueryable<T> / SelectBuilder  │
│  WindowFunction / Pager / BatchFinder             │
├─────────────────────────────────────────────────┤
│              实体运行时层 (Entity Runtime)          │
│  EntitySession / EntityFactory / EntityOperate    │
│  EntityCache / SingleCache / FieldCache           │
│  EntityTransaction / DirtyCollection              │
│  IEntityInterceptor / NavigationLoader            │
│  EntityQueue / EntityDeferredQueue                │
├─────────────────────────────────────────────────┤
│          数据访问层 (DAL / Data Access Layer)      │
│  DAL / IDatabase / IDbSession / ConnectionPool    │
│  DbFactory / DriverLoader                          │
│  SelectBuilder / InsertBuilder / DbPackage        │
│  ReadWriteStrategy / SaveModes                    │
├─────────────────────────────────────────────────┤
│          数据库提供者层 (Database Providers)        │
│  SQLite / MySQL / SqlServer / Oracle / PG         │
│  达梦 / 金仓 / 瀚高 / DB2 / Hana / IRIS / VastBase │
│  TDengine / InfluxDB / ClickHouse / DuckDB        │
│  MongoDB / Network / NovaDb / Access¹ / SqlCe¹    │
└─────────────────────────────────────────────────┘
```

> ¹ Access 和 SqlCe 仅限 `NETFRAMEWORK`，当前未在 `DbFactory` 注册。

## 3. 关键流程

### 3.1 实体查询流程

```
Entity<T>.FindAll(whereExpression)
    → EntitySession.Query(where)
        → 尝试 Meta.Count 行数缓存（快速返回）
        → 尝试 EntityCache 整表缓存（<1000行）
        → 尝试 SingleCache 单对象缓存（主键查询）
        → 尝试 DbCache 数据层缓存（SQL 查询结果）
        → DAL.Session.Query(sql) 实际数据库查询
            → ConnectionPool.GetConnection()
            → DbCommand.ExecuteReader()
            → 实体映射 (EntityFactory.CreateEntity)
        → 写入各层缓存
```

### 3.2 实体写入流程

```
Entity.Insert/Update/Delete
    → IEntityInterceptor.OnXxx (AOP 前置钩子)
    → DirtyCollection 追踪变更字段
    → EntityTransaction 事务管理（如开启事务）
    → EntitySession.Insert/Update/Delete
        → DAL.Session.Execute(sql)
        → 写操作缓存失效/更新
    → IEntityInterceptor.OnXxx (AOP 后置钩子)
    → EntityQueue 异步批写（如启用）
```

### 3.3 自动迁移流程

```
应用启动 → Meta.Build(迁移模式)
    → EntityFactory.Init(entityType)
        → 加载 XML 模型 / Attribute 元数据
        → CheckModel() 校验模型一致性
        → CheckTable() 检查表结构
            → Migration.On: 自动建表/加字段/建索引
            → Migration.ReadOnly: 仅检查不修改
            → Migration.Off: 跳过
```

## 4. 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 实体模型 | 充血模型 | 行为与数据不分离，Biz 分部类可扩展业务逻辑 |
| 缓存策略 | 多级阶梯缓存 | 读多写少场景优化，行数/整表/单对象/数据层四级递进 |
| 查询 DSL | 链式 WhereExpression + IQueryable LINQ | 既保留强类型链式 API，又兼容标准 LINQ 生态 |
| 模型驱动 | XML + Attribute 双通道 | XML 适合代码生成，Attribute 适合运行时元数据 |
| 分片策略 | 策略模式 (IShardPolicy) | 可扩展时间/哈希/范围等分片策略 |
| 数据库驱动 | 工厂模式 (DbFactory) | 按 DatabaseType 注册，按需创建驱动实例 |
| 反向工程 | 模板方法 (DbMetaData) | 正向（DDL 生成）和反向（Schema 读取）统一抽象 |
| 服务层 | DbService 四层架构 | 支持本地注入和 HTTP 远程调用两种部署模式 |

## 5. 模块依赖关系

```
Entity (最上层，面向用户)
  ├── Model (查询表达式)
  ├── Cache (多级缓存)
  ├── Configuration (元数据配置)
  ├── DAL (数据访问抽象)
  │   ├── Database (数据库驱动)
  │   ├── Common (连接池/工厂/会话)
  │   └── MetaData (元数据/反向工程)
  ├── Code (代码生成器) — 开发时工具
  ├── Membership (RBAC 权限)
  ├── Services (HTTP 数据库服务)
  ├── Statistics (统计聚合)
  ├── Shards (分表分库)
  ├── Tree (树形实体)
  ├── Transform (ETL 抽取器)
  ├── Linq (IQueryable 支持)
  ├── Attributes (实体特性)
  └── Common (通用工具)
```

## 7. 待补充流程

以下关键流程需要人工补充序列图或详细说明：

- **DbService 远程调用序列**：客户端 → HTTP → DbController → DbServer → DbService → DAL → Database
- **二级缓存分布式协调**：应用 A 写操作 → CacheInvalidator → Redis 发布 → 应用 B 收到失效通知 → 清除本地缓存
- **EntityQueue 异步批写**：多线程 Insert → EntityQueue 缓冲 → 定时 Flush → 批量 SQL Merge
- **跨分片查询 (AutoShard)**：查询路由 → 各分片并行查询 → 结果聚合 → 排序/分页
