# NewLife.XCode 竞品分析报告

## 1. 概述

本报告对比 **NewLife.XCode**（数据中间件 / 超级 ORM）与当前 .NET 生态中主流 ORM/数据访问库，从功能覆盖、性能、依赖、许可证、社区活跃度等维度进行综合评估。

XCode 定位：高性能 .NET 数据中间件，聚焦"添删改查 + 极致性能 + 海量数据"，内置多级缓存、自动建模/迁移、分表分库、强类型查询、跨库迁移，单表生产实践达百亿级。

### 5.2a API 易用性对比：竞品代码示例

以下通过具体代码对比 XCode 与竞品在关键差距上的 API 表达差异。

#### LINQ 查询支持：动态条件拼接的两种思路

XCode 选择 `WhereExpression` 的核心原因正是用户提到的——**实际业务中大量场景需要条件不存在时不拼接**。这个需求在 LINQ 体系中通过 `IQueryable<T>` 组合查询满足：

```csharp
// 🔴 XCode：WhereExpression 链式拼接（2005 年设计选择）
// 针对"有条件时就拼接，没条件时不拼接"场景，WhereExpression 最直观
var exp = new WhereExpression();
if (!name.IsNullOrEmpty()) exp &= _.UserName.Contains(name);
if (age > 0) exp &= _.Age == age;
if (deptId > 0) exp &= _.DeptId == deptId;
return UserInfo.FindAll(exp, pager);

// 🟢 竞品方案一：EF Core / linq2db - IQueryable 零开销组合
// 每次 Where 返回新 IQueryable，不执行 SQL，最终 ToList 才生成完整 SQL
IQueryable<User> query = ctx.Users;
if (!string.IsNullOrEmpty(name))
    query = query.Where(u => u.UserName.Contains(name));
if (age > 0)
    query = query.Where(u => u.Age == age);
if (deptId > 0)
    query = query.Where(u => u.DeptId == deptId);
var list = query.OrderByDescending(u => u.Id)
    .Skip((page - 1) * size).Take(size)
    .ToList();

// 🟢 竞品方案二：FreeSql / SqlSugar - WhereIf 内联条件判断
// 框架内置 WhereIf(条件, 表达式)，条件不满足时自动忽略该条件
var list = fsql.Select<User>()
    .WhereIf(!name.IsNullOrEmpty(), u => u.UserName.Contains(name))
    .WhereIf(age > 0, u => u.Age == age)
    .WhereIf(deptId > 0, u => u.DeptId == deptId)
    .OrderByDescending(u => u.Id)
    .Page(page, size)
    .ToList();

// 🟢 竞品方案三：EF Core 单表达式 + 短路求值（最不推荐）
// 利用 && 短路特性，条件不满足时 OrDefault 返回 true 使整个条件失效
// 但所有参数仍会传递给数据库，可能影响查询计划
var list = ctx.Users.Where(u =>
    (string.IsNullOrEmpty(name) || u.UserName.Contains(name)) &&
    (age <= 0 || u.Age == age) &&
    (deptId <= 0 || u.DeptId == deptId)
).ToList();
```

> **设计取舍分析**：WhereExpression 是 2005 年 XCode 创立时的设计，当时 .NET 2.0 时代尚无 LINQ，Expression 表达式树也未诞生。WhereExpression 的优势是**直观、对 SQL 开发者友好、条件组合完全可控**；LINQ 方案的优势是**编译期检查、IDE 智能感知、重构安全**。两者表达能力等价，FreeSql/SqlSugar 的 `WhereIf` 在 API 直觉上最接近 WhereExpression 体验。

#### LINQ 最终目标方案（确定实现）

XCode 已确定支持 LINQ，同时提供 IQueryable&lt;T&gt; 和 WhereIf 两套 API，与现有 WhereExpression 共存：

```csharp
// ✅ 方案一：IQueryable<T> 组合查询（EF Core 模式）
// 每次 Where 返回新 IQueryable，零开销组合，最终 ToList 才生成 SQL
IQueryable<User> query = UserInfo.Query;
if (!name.IsNullOrEmpty())
    query = query.Where(u => u.UserName.Contains(name));
if (age > 0)
    query = query.Where(u => u.Age == age);
if (deptId > 0)
    query = query.Where(u => u.DeptId == deptId);
var list = query.OrderByDescending(u => u.Id)
    .Skip((page - 1) * size).Take(size)
    .ToList();

// ✅ 方案二：WhereIf 内联条件（XCode 2010 年即已支持，本次增强）
var list = UserInfo.FindAllWhereIf(
    (!name.IsNullOrEmpty(), _.UserName.Contains(name)),
    (age > 0, _.Age == age),
    (deptId > 0, _.DeptId == deptId),
    pager);

// ✅ 方案三：原有 WhereExpression（保持兼容，不加 LINQ 的项目继续用）
var exp = new WhereExpression();
if (!name.IsNullOrEmpty()) exp &= _.UserName.Contains(name);
if (age > 0) exp &= _.Age == age;
var list = UserInfo.FindAll(exp, pager);
```

**条件编译策略**：IQueryable&lt;T&gt; 体系（Expression/Queryable）在 net45/net461 中存在部分限制，用条件编译隔离：

```csharp
#if NET6_0_OR_GREATER
    // IQueryable<T> 完整支持 .NET 6+
    public static IQueryable<T> Query { get; }
#else
    // net45/net461 仅提供 WhereIf 方案
    // IQueryable 不可用，降级到 WhereExpression
#endif
```

> **最终结论**：三套 API 共存——IQueryable（现代 LINQ 体验）+ WhereIf（XCode 经典快速拼接）+ WhereExpression（传统链式）。不同 .NET 版本自动降级，用户无感。

#### 导航属性 / 关联查询

XCode 的"内存联表"策略 vs 竞品的导航属性：

```csharp
// 🔴 XCode：内存联表（先查缓存，再内存关联）
var users = UserInfo.FindAllWithCache();       // 整表缓存命中
var orders = Order.FindAllWithCache();          // 整表缓存命中
// 内存中做关联（伪联表）
var list = users.Join(orders, u => u.Id, o => o.UserId,
    (u, o) => new { User = u, Order = o });

// 🟢 EF Core / FreeSql：导航属性（延迟加载 / Include）
// 实体定义
class User {
    public Int32 Id { get; set; }
    public String Name { get; set; }
    public List<Order> Orders { get; set; }  // 导航属性
}
// 查询
var users = ctx.Users.Include(u => u.Orders)
    .Where(u => u.Age > 18)
    .ToList();
// 访问：user.Orders 自动加载或已包含

// 🟢 SqlSugar：导航属性
var list = db.Queryable<User>()
    .Includes(u => u.Orders)
    .Where(u => u.Age > 18).ToList();
```

> 导航属性优势：代码表达直观，框架自动处理关联 SQL。XCode 的内存联表在缓存命中时性能极佳，但在缓存未命中或全表过大时不适用。两者各有适用场景。

#### Fluent API 配置

XCode 的 XML 模型 + Attribute 方式 vs 竞品的纯代码配置：

```csharp
// 🔴 XCode：XML 模型 + Attribute
// db.xml
// <Table Name="UserInfo"><Columns>
//   <Column Name="Id" DataType="Int32" Identity="True" PrimaryKey="True" />
// </Columns></Table>
//
// 实体类
[Model("db.xml")]
public class UserInfo : Entity { }

// 🟢 EF Core Fluent API
class MyDbContext : DbContext {
    public DbSet<User> Users { get; set; }
    protected override void OnModelCreating(ModelBuilder model) {
        model.Entity<User>(e => {
            e.ToTable("UserInfo");
            e.HasKey(u => u.Id);
            e.Property(u => u.Name).IsRequired().HasMaxLength(50);
            e.HasMany(u => u.Orders).WithOne(o => o.User).HasForeignKey(o => o.UserId);
        });
    }
}

// 🟢 FreeSql Fluent API
fsql.CodeFirst.ConfigEntity<User>(e => {
    e.Property(u => u.Name).IsRequired().HasMaxLength(50);
});
// 或使用 Attribute
class User {
    [Column(IsIdentity = true, IsPrimary = true)]
    public Int32 Id { get; set; }
    [Column(StringLength = 50, IsNullable = false)]
    public String Name { get; set; }
}
```

> Fluent API 优势：无需外部文件，纯 C# 配置，编译期检查，动态调整灵活。XCode 的 XML 模型驱动在多人协作和大规模实体管理时更结构化，但缺少 IDE 智能感知。

## 6. 结论与建议

### 6.1 定位总结

XCode 在 .NET ORM 市场中具有**独特定位**：

- **不是又一个通用 ORM**，而是"数据中间件" — 集 ORM + 缓存 + 分片 + ETL + 同步 + 配置中心于一体
- **缓存优先架构**是核心差异化优势，在读多写少场景下的性能远超任何竞品
- **国产替代首选**：对国产数据库和时序数据库的支持领先同类，MIT 协议商业友好

### 6.2 适用场景推荐

| 场景 | 推荐 | 理由 |
|------|:---:|------|
| 读多写少、高并发查询 | ⭐⭐⭐ XCode | 多级缓存命中率极高 |
| 海量数据（亿级以上） | ⭐⭐⭐ XCode | 分表分库 + 分页优化 |
| 国产数据库项目 | ⭐⭐⭐ XCode | 国产库适配最全面 |
| 复杂多表关联查询 | ⭐⭐ EF Core / linq2db | LINQ 表达力更强 |
| 极简场景、已有 SQL | ⭐⭐ Dapper | 零依赖、极致轻量 |
| 需要 AOT 发布 | ⭐⭐ FreeSql / SqlSugar | XCode 明确不支持 AOT |
| 企业级数据平台 | ⭐⭐⭐ XCode | ETL + 同步 + 配置中心 |

### 6.3 追赶优先级建议

| 优先级 | 事项 | 说明 |
|--------|------|------|
| **✅ 已完成** | LINQ 查询支持（IQueryable + WhereIf） | `EntityQueryable<T>` + `EntityQueryProvider` + `LinqExpressionVisitor` |
| **✅ 已完成** | ClickHouse / DuckDB / MongoDB Provider 适配 | 新增三个 Provider，覆盖列存/嵌入式OLAP/文档数据库 |
| **✅ 已完成** | 窗口函数 / CTE | `SelectBuilder.Window` + `WindowFunction` 助手 |
| **✅ 已完成** | 二级缓存（分布式） | `XCodeSetting.CacheProvider` + `CacheInvalidator` 版本号机制 |
| **⚠️ 部分完成** | 导航属性/关联查询（Include） | `Include(Type)` 预加载关联实体缓存 |
| **P2（长期）** | 英文文档完善、社区推广 | 扩大国际影响力 |
| **❌ 不做** | AOT/NativeAOT 支持 | 深度依赖反射和动态生成 |
| **❌ 不做** | 可视化模型设计器 | XML 模型编辑已有 XSD 智能感知 |
| **🧊 暂缓** | Fluent API 配置 | XML 模型 + Attribute 已覆盖实体配置需求 |
