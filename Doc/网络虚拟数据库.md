# 网络虚拟数据库（Network）需求与架构设计

## 一、需求概述

XCode 曾经支持网络虚拟数据库 `Network`（DatabaseType=100），通过网络接口将数据库操作转发到远端服务执行。现需恢复并重构该功能，改用 HTTP 协议承载，基于 `ApiHttpClient` / `HttpServer` 实现。

### 1.1 核心目标

| 目标 | 说明 |
|------|------|
| **透明代理** | 应用层使用标准 XCode 实体操作，底层自动转为 HTTP 接口调用 |
| **简单可靠** | 不追求极致性能，面向普通场合（管理后台、数据同步、跨服务查询等） |
| **灵活部署** | 服务端可独立启动（DbServer），也可嵌入 ASP.NET 应用（注入 DbService） |

### 1.2 连接字符串格式

```
Server=http://127.0.0.1:3305;Database=Membership;Password=token123;Provider=Network
```

| 参数 | 说明 |
|------|------|
| `Server` | 服务端 HTTP 接口地址，支持 `ApiHttpClient` 的多地址负载均衡格式 |
| `Database` | 目标数据库连接名（服务端 DAL 中已注册的连接名） |
| `Password` | Token 令牌，用于身份验证 |
| `Provider` | 固定为 `Network` |

### 1.3 功能清单

| 编号 | 功能 | 说明 |
|------|------|------|
| F1 | 登录认证 | 客户端携带 database + token，服务端验证并返回数据库信息 |
| F2 | SQL 查询 | 传递 SQL + 参数，返回 DbTable 结果集（JSON 序列化） |
| F3 | SQL 执行 | 传递 SQL + 参数，返回受影响行数 |
| F4 | 插入并获取自增ID | 传递 SQL + 参数，返回新增行自增编号 |
| F5 | 快速查询记录数 | 传递表名，返回近似记录数 |
| F6 | 获取表结构 | 获取服务端数据库的表结构元数据 |

## 二、架构设计

### 2.1 整体架构

```
┌─────────────────────────────────────────────────────┐
│                   应用层 (Entity)                     │
│           标准 XCode 实体 CRUD 操作                    │
└────────────────────┬────────────────────────────────┘
                     │
┌────────────────────▼────────────────────────────────┐
│            Network (虚拟数据库驱动)                     │
│   NetworkSession / NetworkMetaData                   │
│   将数据库操作转为 DbClient HTTP 调用                   │
└────────────────────┬────────────────────────────────┘
                     │ HTTP (ApiHttpClient)
┌────────────────────▼────────────────────────────────┐
│              DbClient (HTTP 客户端)                    │
│   封装 ApiHttpClient，提供 Query/Execute 等方法        │
└────────────────────┬────────────────────────────────┘
                     │ HTTP Request/Response
┌────────────────────▼────────────────────────────────┐
│         DbController (HTTP 控制器)                    │
│   /Db/Login、/Db/Query、/Db/Execute 等接口            │
│   参数校验 + 转发给 DbService                         │
└────────────────────┬────────────────────────────────┘
                     │
┌────────────────────▼────────────────────────────────┐
│             DbService (业务服务层)                     │
│   实际执行 DAL 操作，可被 DbServer 和 ASP.NET 共用     │
└────────────────────┬────────────────────────────────┘
                     │
┌────────────────────▼────────────────────────────────┐
│         DAL → 实际数据库 (MySQL/SQLite/...)           │
└─────────────────────────────────────────────────────┘
```

### 2.2 目录结构（Service → Services）

```
XCode/
  Services/
    DbClient.cs          # HTTP 客户端，封装 ApiHttpClient
    DbController.cs      # HTTP 控制器，提供 REST 接口
    DbServer.cs           # HTTP 服务器，继承 HttpServer
    DbService.cs          # 业务服务层，实际执行 DAL 操作
    LoginInfo.cs          # 登录信息模型
  DataAccessLayer/
    Common/
      DatabaseType.cs     # 增加 Network = 100
    Database/
      Network.cs          # 网络虚拟数据库驱动
```

### 2.3 类职责

| 类 | 职责 |
|------|------|
| `DbService` | 核心业务逻辑：接收数据库名+SQL+参数，调用 DAL 执行，返回结果。令牌验证。支持注入到 DbServer 和 ASP.NET 控制器 |
| `DbClient` | 封装 `ApiHttpClient`，提供 `LoginAsync`/`QueryAsync`/`ExecuteAsync` 等异步方法。处理 Token 认证、参数序列化 |
| `DbController` | HTTP 控制器（POCO），路由 `/Db/*`，参数校验后转发给 `DbService` |
| `DbServer` | 继承 `HttpServer`，启动 HTTP 监听，注册 `DbController`，注入 `DbService` |
| `Network` | XCode 虚拟数据库驱动，实现 `DbBase`，内部持有 `DbClient` 实例 |
| `NetworkSession` | 数据库会话，重写 `Query`/`Execute` 等方法，调用 `DbClient` |
| `NetworkMetaData` | 元数据，通过 HTTP 获取远端表结构 |
| `LoginInfo` | 登录响应模型：DbType、Version |

### 2.4 HTTP 接口定义

| 方法 | 路径 | 请求参数 | 响应 |
|------|------|---------|------|
| POST | `/Db/Login` | `{ db, token }` | `LoginInfo { DbType, Version }` |
| POST | `/Db/Query` | `{ sql, parameters? }` | `DbTable` (JSON) |
| POST | `/Db/Execute` | `{ sql, parameters? }` | `{ rows: Int64 }` |
| POST | `/Db/InsertAndGetIdentity` | `{ sql, parameters? }` | `{ id: Int64 }` |
| GET  | `/Db/QueryCount` | `?tableName=xxx` | `{ count: Int64 }` |
| GET  | `/Db/GetTables` | 无 | `IDataTable[]` (JSON) |

### 2.5 认证流程

1. 客户端在连接字符串中设置 `Password=token123`
2. `DbClient` 构造时将 Token 设置到 `ApiHttpClient.Token`
3. 首次操作时自动调用 `/Db/Login`，传递 `database` 和 `token`
4. 服务端 `DbService.Login` 验证 Token，创建 DAL 实例
5. 后续请求通过 HTTP Header `Authorization: Bearer token123` 认证

### 2.6 数据序列化

- **请求参数**：使用 JSON 序列化（ApiHttpClient 内置支持）
- **查询结果**：DbTable 通过 JSON 序列化传输（列名 + 行数据）
- **表结构**：IDataTable 序列化为 JSON
