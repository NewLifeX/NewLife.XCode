# NewLife.XCode 需求文档

> 版本：v12.0 | 日期：2026-07-15

本文档描述 NewLife.XCode 的愿景、核心目标和模块划分。完成状态在[功能模块清单](功能模块清单.md)中追踪，竞品对比见[竞品分析报告](竞品分析报告.md)，详细设计在各模块架构文档中展开。

模块按**前置依赖原则**排序：底层基础模块在前，上层应用模块在后。

## 1. 背景与愿景

### 1.1 系统定位

高性能 .NET 数据中间件（超级 ORM），聚焦"添删改查 + 极致性能 + 海量数据"，内置多级缓存、自动建模/迁移、分表分库、强类型查询、跨库迁移，支持 19 种关系型/时序型/列存/文档/网络虚拟数据库。单表生产实践达百亿级，查询吞吐可达十亿级 QPS（配合缓存策略）。

### 1.2 愿景

成为 .NET 生态中最快速、最稳定、最易用的数据中间件——"越用越快"：写入越多，缓存命中越精准；查询越多，预热越充分。为物联网、工业互联网、金融交易等高吞吐场景提供坚实的数据底座。

### 1.3 设计原则

- **性能优先**：核心主路径多级缓存 + 批量优化，查询速度远超直接写 SQL
- **广泛兼容**：net45 / net461 / netstandard2.0 / netstandard2.1 / net6.0+ / net8.0+ / net10.0+
- **国产化就绪**：全面支持国产数据库和国产操作系统
- **MIT 许可证**：完全开放，允许商用/修改/再发行，可替换品牌名实现私有化

## 2. M1 数据库支持

数据库支持是 XCode 的基石，提供统一抽象层（DAL + IDatabase + IDbSession + ConnectionPool），屏蔽各数据库差异，支持按 DatabaseType 注册工厂。

- **M1-1 SQL Server**：分页算法优化，支持 SQL Server 各版本
- **M1-2 MySQL / MariaDB**：完整支持
- **M1-3 PostgreSQL**：完整支持
- **M1-4 Oracle**：完整支持
- **M1-5 SQLite**：默认开发库，零配置
- **M1-6 达梦 (DM)**：国产数据库支持
- **M1-7 人大金仓 (Kingbase)**：国产数据库支持
- **M1-8 瀚高 (HighGo)**：国产数据库支持
- **M1-9 DB2**：企业级支持
- **M1-10 TDengine**：时序数据库支持
- **M1-11 InfluxDB**：时序数据库支持
- **M1-12 ClickHouse**：列式分析型数据库，用于 OLAP 场景
- **M1-13 DuckDB**：嵌入式 OLAP 数据库
- **M1-14 MongoDB**：文档型 NoSQL 数据库
- **M1-15 Hana (SAP HANA)**：企业级内存计算数据库，支持 OLAP/OLTP 混合负载
- **M1-16 IRIS (InterSystems IRIS)**：高性能多模型事务分析型数据库
- **M1-17 VastBase**：基于 PostgreSQL 的国产数据库
- **M1-18 Network（网络虚拟数据库）**：通过 HTTP 接口将数据库操作转发到远端服务执行
- **M1-19 NovaDb**：NewLife 自研数据库，支持嵌入模式和网络模式

## 3. M2 模型驱动与代码生成

定义数据结构是使用 XCode 的第一步，支持 XML 模型和 Attribute 两种元数据通道，提供代码生成器、反向工程和自动迁移能力。

- **M2-1 XML 模型驱动**：Tables/Columns/Indexes → 代码生成器 → 数据类 + 业务分部类
- **M2-2 代码生成器 (xcodetool)**：数据类每次覆盖，业务类首次生成保留，避免代码冲突
- **M2-3 反向工程 (DbFirst)**：从已有数据库抽取 Schema 生成实体类
- **M2-4 自动迁移 (CodeFirst)**：新增字段上线自动同步，支持 Off/ReadOnly/On/Full 四级策略
- **M2-5 Attribute 标注**：`[BindTable]`、`[BindColumn]`、`[BindIndex]` 等特性标注实体元数据

## 4. M3 基础添删改查

实体最核心的操作：插入、更新、删除。配合保存模式、事务、拦截器、脏数据跟踪等机制，构成完整的写入能力。

- **M3-1 插入 (Insert)**：单行插入 + 批量插入，自增主键自动填充
- **M3-2 更新 (Update)**：单行更新 + 指定字段更新，批量更新
- **M3-3 删除 (Delete)**：单行删除 + 条件删除，批量删除
- **M3-4 保存模式 (SaveModes)**：Insert / Upsert / InsertIgnore / Replace 四种保存策略
- **M3-5 实体事务 (EntityTransaction)**：事务内批量操作，提交/回滚后统一处理缓存一致性
- **M3-6 实体拦截器 AOP (IEntityInterceptor)**：Insert/Update/Delete 前后钩子，支持日志、审计等横切关注点
- **M3-7 脏数据跟踪 (DirtyCollection)**：实体属性变更追踪，仅更新变更字段，减少 SQL 开销
- **M3-8 实体队列 / 异步批写 (EntityQueue)**：实体变更异步入队，批量合并后落库，适合高吞吐写入场景
- **M3-9 实体延迟队列 (EntityDeferredQueue)**：缓冲合并实体操作，定时 Flush 后批量 SQL Merge，基于 NewLife.Core DeferredQueue

## 5. M4 查询与条件 DSL

XCode 同时提供链式 WhereExpression 和标准 IQueryable\<T\> LINQ 两种查询 API，配合分页/排序/投影/窗口函数等形成完整的查询能力。

- **M4-1 WhereExpression 链式查询**：组合条件 &/| 链式表达式，支持 Contains、Between、Like、In 等
- **M4-2 IQueryable\<T\> LINQ 查询**：EntityQueryable / EntityQueryProvider，支持 Where/OrderBy/Skip/Take/First/Count 等标准操作
- **M4-3 WhereIf 内联条件**：条件满足时拼接 Where，不满足时自动忽略，简化动态查询
- **M4-4 导航属性 / 关联查询 (Include)**：`Include(Type)` 预加载关联实体缓存，配合 NavigationLoader 运行时加载导航属性
- **M4-5 分页与排序 (Pager)**：统一分页参数对象，自动处理 TotalCount 和多字段排序方向
- **M4-6 字段投影与自定义排序**：指定查询列、多字段排序方向
- **M4-7 原生 SQL 查询**：执行任意 SQL 并映射到实体，兼容 Dapper 用法
- **M4-8 多结果集**：一次查询返回多个结果集
- **M4-9 内存联表**：先命中缓存再内存关联，规避复杂 JOIN，XCode 独有策略
- **M4-10 窗口函数**：ROW_NUMBER / RANK / DENSE_RANK 及聚合窗口函数，SelectBuilder.Window + WindowFunction 助手类
- **M4-11 DAL.Select\<T\>() 流式查询**：DAL 实例方法返回 IQueryable，自动绑定连接，支持 WhereIf/Page 流式调用
- **M4-12 导航属性 Fluent API**：`HasOne\<TTarget\>()` / `HasMany\<TTarget\>()` 链式注册导航关系

## 6. M5 多级缓存体系

针对"读多写少"场景设计的多级阶梯缓存，从行数→整表→单对象→数据层四级递进，配合过期刷新、写操作失效、二级分布式协调等机制。

- **M5-1 Meta.Count 行数缓存**：快速行数缓存，>1000 行走系统索引
- **M5-2 EntityCache 整表缓存**：适合小表（<1000 行），全量缓存到内存
- **M5-3 SingleCache 单对象缓存**：主键/从键字典级缓存，适合频繁点查
- **M5-4 DbCache 数据层缓存**：DAL 层 SQL 查询结果缓存
- **M5-5 缓存过期与刷新策略**：默认 60s 过期，过期返回旧数据 + 异步刷新
- **M5-6 写操作缓存一致性**：写操作即时失效/更新，事务中累积提交后统一处理
- **M5-7 二级缓存（分布式）**：基于 Redis 的跨进程版本号协调机制（CacheInvalidator.Provider）
- **M5-8 字段缓存 (FieldCache)**：基于 EntityCache 的字段值统计缓存，自动分组计数，适合下拉选项等场景

## 7. M6 分表分库

当单表数据量达到千万/亿级时，通过分片策略将数据分布到多个表/库。支持时间、主键、哈希等策略，业务层代码透明无感。

- **M6-1 ShardPolicy 抽象**：统一的分片策略接口 IShardPolicy
- **M6-2 TimeShardPolicy 时间分片**：按日期自动生成表名 Table_yyyyMMdd
- **M6-3 主键/哈希分片**：按主键范围或哈希值路由
- **M6-4 分片透明路由**：业务层代码不变，自动路由到对应分片
- **M6-5 跨分片查询 (AutoShard)**：跨多个分片的聚合查询

## 8. M7 高级数据中间件

超越 ORM 核心定位的高级特性：ETL 抽取器、备份恢复、数据库 HTTP 服务、统计聚合、树形实体、数据权限等。

- **M7-1 ETL 数据抽取**：数据抽取器（EntityIdExtracter/IdExtracter/PagingExtracter/TimeExtracter），支持 ExtractSetting 配置参数；完整 ETL 管道引擎已移除
- **M7-3 数据备份与恢复**：数据库级别备份恢复，支持跨库同步（DAL_Backup + DbPackage 数据包）
- **M7-4 树形实体 (EntityTree)**：支持树结构实体操作的基础类 EntityTree\<TEntity\>
- **M7-5 数据权限**：行级数据权限过滤，DataScopeModule/IDataScope 多租户隔离
- **M7-6 统计与聚合扩展**：同比环比、聚合查询扩展，StatHelper/StatModel/StatLevels
- **M7-7 数据库 HTTP 服务 (DbService)**：四层架构（DbClient/DbController/DbServer/DbService），通过 HTTP 远程执行数据库操作，支持令牌验证
- **M7-8 数据模拟压测 (DataSimulation)**：多线程并发写入模拟，支持事务批大小控制
- **M7-9 批量主键查找器 (BatchFinder)**：批量送入主键后逐个查询，自动缓存结果，批大小可配
- **M7-10 多租户支持**：ITenantSource/TenantTypes，与 DataScope 配合实现租户隔离

## 10. 不做什么

| 编码 | 功能 | 理由 |
|:----:|------|------|
| N1 | AOT/NativeAOT 支持 | 深度依赖反射和动态代码生成，AOT 裁剪会破坏核心架构 |
| N2 | 可视化模型设计器 | 性价比不高，XML 模型编辑已有 XSD 架构提供智能感知 |
| N3 | NoSQL 原生查询 | MongoDB 等仅作 Provider 接入，不提供非关系型原生查询 API |
| N4 | GraphQL 支持 | 超出数据中间件定位范围 |
| N5 | 微服务/服务网格 | 由 NewLife.Stardust 生态提供 |
