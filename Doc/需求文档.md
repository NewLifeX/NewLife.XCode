# NewLife.XCode 需求文档

> 版本：v12.0 | 日期：2026-07-15

本文档描述 NewLife.XCode 的愿景、核心目标和模块划分。完成状态在[功能模块清单](功能模块清单.md)中追踪，竞品对比见[竞品分析报告](竞品分析报告.md)，详细设计在各模块架构文档中展开。

## 1. 背景与愿景

### 1.1 系统定位

高性能 .NET 数据中间件（超级 ORM），聚焦"添删改查 + 极致性能 + 海量数据"，内置多级缓存、自动建模/迁移、分表分库、强类型查询、跨库迁移，支持 19 种关系型/时序型/列存/文档/网络虚拟数据库。单表生产实践达百亿级，查询吞吐可达十亿级 QPS（配合缓存策略）。

### 1.2 愿景

成为 .NET 生态中最快速、最稳定、最易用的数据中间件——"越用越快"：写入越多，缓存命中越精准；查询越多，预热越充分。为物联网、工业互联网、金融交易等高吞吐场景提供坚实的数据底座。

### 1.3 设计原则

- **性能优先**：核心主路径多级缓存 + 批量优化，查询速度远超直接写 SQL
- **广泛兼容**：net45 / net461 / netstandard2.0 / netstandard2.1 / net6.0+ / net8.0+ / net10.0+
- **国产化就绪**：全面支持国产数据库和国产操作系统
- **MIT 许可证**：完全开放，允许商用/修改/再发行，可替换品牌名实现私有化

## 2. M1 查询与条件 DSL

- **M1-1 WhereExpression 链式查询**：组合条件 &/| 链式表达式，支持 Contains、Between、Like、In 等
- **M1-2 IQueryable&lt;T&gt; LINQ 查询**：实体级标准 IQueryable 接口
- **M1-3 WhereIf 内联条件**：条件满足时拼接 Where，不满足时自动忽略
- **M1-4 Include 预加载**：`Include(Type)` 预加载关联实体缓存
- **M1-5 分页与排序**：统一分页参数对象，自动处理 TotalCount 和排序
- **M1-6 字段投影与自定义排序**：指定查询列、多字段排序方向
- **M1-7 原生 SQL 查询**：支持执行任意 SQL 并映射到实体
- **M1-8 多结果集**：一次查询返回多个结果集
- **M1-9 内存联表**：先命中缓存再内存关联，规避复杂 JOIN
- **M1-10 窗口函数**：ROW_NUMBER / RANK / DENSE_RANK 及聚合窗口函数
- **M1-11 DAL.Select&lt;T&gt;() 流式查询入口**：自动绑定连接，支持 WhereIf/Page 流式调用
- **M1-12 导航属性 Fluent API**：`HasOne<TTarget>()` / `HasMany<TTarget>()` 链式注册导航关系，配合 NavigationLoader 运行时加载导航属性

## 3. M2 多级缓存体系

- **M2-1 Meta.Count 行数缓存**：快速行数缓存，>1000 行走系统索引
- **M2-2 EntityCache 整表缓存**：适合小表（<1000 行），全量缓存到内存
- **M2-3 SingleCache 单对象缓存**：主键/从键字典级缓存
- **M2-4 DbCache 数据层缓存**：DAL 层 SQL 查询结果缓存
- **M2-5 缓存过期与刷新策略**：默认 60s 过期，过期返回旧数据 + 异步刷新
- **M2-6 写操作缓存一致性**：写操作即时失效/更新，事务中累积提交后统一处理
- **M2-7 二级缓存（分布式）**：基于 Redis 的跨进程版本号协调机制
- **M2-8 字段缓存 (FieldCache)**：基于 EntityCache 的字段值统计缓存，自动分组计数，适合下拉选项等场景

## 4. M3 模型驱动与代码生成

- **M3-1 XML 模型驱动**：Tables/Columns/Indexes → 代码生成器 → 数据类 + 业务分部类
- **M3-2 代码生成器 (xcodetool)**：数据类每次覆盖，业务类首次生成保留
- **M3-3 反向工程 (DbFirst)**：从已有数据库抽取模型生成实体
- **M3-4 自动迁移 (CodeFirst)**：新增字段上线自动同步，无需额外 SQL 脚本
- **M3-5 Attribute 标注**：`[Model]` `[BindIndex]` 等特性标注实体元数据

## 5. M4 分表分库

- **M4-1 ShardPolicy 抽象**：统一的表/库分片策略接口
- **M4-2 TimeShardPolicy 时间分片**：按日期自动生成表名 Table_yyyyMMdd
- **M4-3 主键/哈希分片**：按主键范围或哈希值路由
- **M4-4 分片透明路由**：业务层代码不变，自动路由到对应分片
- **M4-5 跨分片查询 (AutoShard)**：跨多个分片的聚合查询

## 6. M5 数据库支持

- **M5-1 SQL Server**：分页算法优化，支持 SQL Server 各版本
- **M5-2 MySQL / MariaDB**：完整支持
- **M5-3 PostgreSQL**：完整支持
- **M5-4 Oracle**：完整支持
- **M5-5 SQLite**：默认开发库，零配置
- **M5-6 达梦 (DM)**：国产数据库支持
- **M5-7 人大金仓 (Kingbase)**：国产数据库支持
- **M5-8 瀚高 (HighGo)**：国产数据库支持
- **M5-9 DB2**：企业级支持
- **M5-10 TDengine**：时序数据库支持
- **M5-11 InfluxDB**：时序数据库支持
- **M5-12 ClickHouse**：列式分析型数据库，用于 OLAP 场景
- **M5-13 DuckDB**：嵌入式 OLAP 数据库
- **M5-14 MongoDB**：文档型 NoSQL 数据库
- **M5-15 Hana (SAP HANA)**：企业级内存计算数据库，支持 OLAP/OLTP 混合负载
- **M5-16 IRIS (InterSystems IRIS)**：高性能多模型事务分析型数据库
- **M5-17 VastBase**：基于 PostgreSQL 的国产数据库
- **M5-18 Network（网络虚拟数据库）**：通过 HTTP 接口将数据库操作转发到远端服务执行
- **M5-19 NovaDb**：NewLife 自研数据库，支持嵌入模式和网络模式

## 7. M6 高级数据中间件

- **M6-1 ETL 数据管道**：抽取、转换、加载全流程管道。抽取器族（EntityIdExtracter/IdExtracter/PagingExtracter/TimeExtracter）已编译可用
- **M6-2 数据同步 (Sync)**：跨库数据同步框架（ISyncMaster/ISyncSlave/SyncManager）
- **M6-3 数据备份与恢复**：数据库级别备份恢复，支持跨库同步（DAL_Backup + DbPackage 数据包）
- **M6-4 实体队列 / 异步批写**：实体变更异步队列，批量合并写入
- **M6-5 实体拦截器 (AOP)**：插入/更新/删除前后钩子
- **M6-6 脏数据跟踪**：实体属性变更追踪
- **M6-7 树形实体**：支持树结构实体操作
- **M6-8 数据权限**：行级数据权限过滤，多租户隔离
- **M6-9 统计与聚合扩展**：同比环比、聚合查询扩展
- **M6-10 数据库HTTP服务 (DbService)**：四层架构（DbClient/DbController/DbServer/DbService），通过 HTTP 远程执行数据库操作，支持令牌验证
- **M6-11 数据模拟压测 (DataSimulation)**：多线程并发写入模拟，支持事务批大小控制
- **M6-12 批量主键查找器 (BatchFinder)**：批量送入主键后逐个查询，自动缓存结果，批大小可配
- **M6-13 实体延迟队列 (EntityDeferredQueue)**：缓冲合并实体操作，批量延迟处理
- **M6-14 多租户支持**：ITenantSource/TenantTypes，与 DataScope 配合实现租户隔离
- **M6-15 保存模式 (SaveModes)**：Insert/Upsert/InsertIgnore/Replace 四种保存模式

## 9. 不做什么

- **AOT/NativeAOT 支持**：XCode 深度依赖反射（实体工厂、元数据缓存、动态属性访问器）和动态代码生成（代码生成器 xcodetool），AOT 裁剪/预编译会严重破坏核心架构。保持运行时动态能力优先，**明确不作为追赶目标**。
- **可视化模型设计器**：经评估不复杂但性价比不高。XML 模型编辑已有 XSD 架构（`Model202604.xsd`）提供智能感知，设计器带来的边际收益有限。**明确不做**。
- **NoSQL 原生查询**：MongoDB 等 NoSQL 数据库仅作为可选 Provider 接入，不提供非关系型原生查询 API
- **GraphQL 支持**：超出数据中间件定位范围
- **微服务/服务网格**：不内置服务发现、负载均衡等，由 NewLife.Stardust 生态提供
